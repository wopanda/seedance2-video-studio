# Seedance 2.0 Video Studio

一个面向 **短视频生成** 的 Seedance 2.0 skill。  
它的目标不是让用户研究参数，而是让用户直接说：

- 想做什么视频
- 有没有参考图
- 想不想重做一版

然后系统帮用户完成：

- 理解需求
- 判断生成路径
- 补默认参数
- 发起任务
- 查状态
- 下载结果
- 回传到聊天

---

## 这是什么

这是一个 **短视频生成 skill**，不是长视频剪辑器。

它目前最适合做：
- 文生短视频
- 1 张图生短视频
- 2 张图做过渡短视频

它当前**不适合直接承诺**：
- 自动做 20 秒 / 30 秒 / 1 分钟长视频
- 自动拆段
- 自动拼接
- 自动续上一条视频
- 完整分镜流水线

一句话：

> **现在它是“短视频生成器”，不是“长视频生产线”。**

---

## 它适合谁

适合这几类人：
- 想直接做一条短视频，不想研究模型参数的人
- 手里有图，想快速做一条动态视频的人
- 已经生成过一条，想再来一版 / 换个感觉的人
- 想把视频生成能力接进聊天工作流的人

---

## 现在支持什么

### 1）只有文字
**文字 → 视频**

比如：
- 做一个 6 秒竖屏广告，鱼缸在现代客厅里，镜头轻推，质感高级。
- 做一个 5 秒电影感短片，雨夜街头，女生慢慢回头看镜头。

---

### 2）文字 + 1 张图
**1 张图（首帧） + 文字 → 视频**

比如：
- 按这张图做一个人物转头视频。
- 保持这张图的人物和场景，做一个轻微镜头运动的视频。

这类本质上是：
**首帧图生视频**。

---

### 3）文字 + 2 张图
**第 1 张图做开头 + 第 2 张图做结尾 → 视频**

比如：
- 用这两张图做一个从 A 过渡到 B 的 6 秒视频。

这类本质上是：
**首尾帧控制的视频生成**。

---

### 4）参考视频 / 参考音频
这块当前还偏技术态。

目前支持方向是：
- 参考视频 URL → 视频生成
- 参考音频 URL → 视频生成

但当前还**不是**：
- 本地视频直接丢进去就全自动跑通
- 本地音频直接丢进去就全自动跑通

所以这部分先不要当成首页主能力来承诺。

---

## 当前不支持什么

请直接按这个理解：

### 暂不支持
- 自动拼长视频
- 自动续段
- 多段自动拆解与重组
- 自动生图再生视频的一条龙产品化流程
- 本地视频 / 本地音频参考的一键产品化路径
- 完整剧本到成片流水线

如果你要做更长的视频，当前更合理的思路是：
- 先拆成几段短片
- 每段分别生成
- 再在后续版本补拼接与统一风格能力

## 用户怎么用

这个 skill 的设计原则是：
**用户说需求，不先说参数。**

### 用法 1：做一个
适合第一次生成。

你可以直接说：
- 做一个 6 秒竖屏广告，展示高端鱼缸，现代客厅场景。
- 做一个 5 秒电影感视频，黄昏街头，男生慢慢走向镜头。

---

### 用法 2：按这个参考做
适合你有图片时。

你可以直接说：
- 按这张图做一个人物转头视频。
- 用这两张图做一个从开场到结尾的过渡视频。

---

### 用法 3：再来一版
适合已经出过结果，想换个感觉。

你可以直接说：
- 再来一版。
- 更像广告一点。
- 镜头更稳一点。
- 保持这个风格，但更高级一点。

---

### 用法 4：问进度
适合任务在跑。

你可以直接说：
- 现在怎么样了？
- 到哪一步了？
- 好了吗？

这类话应该只查进度，**不重新下单**。

---

## 系统会怎么处理

当用户发来一句话后，系统会尽量自动完成：

1. 判断你是：
   - 做一个
   - 按参考做
   - 再来一版

2. 判断素材角色：
   - 首帧
   - 末帧
   - 风格参考
   - 节奏参考

3. 自动补默认参数：
   - duration
   - ratio
   - style
   - negative constraints

4. 自动执行：
   - create
   - wait
   - download
   - 回传结果

只有在这些情况下才追问：
- 多张图主次不清
- 用户想法明显冲突
- 用户明确要求先看方案 / prompt
- 参考素材太多，优先级不清

## 当前最准确的产品边界

### 现在能承诺的
- 短视频生成
- 图生短视频
- 两图过渡短视频
- 任务创建 / 查状态 / 下载
- 基本的结果回传闭环
- 防重复下单

### 现在不能承诺的
- 长视频自动生成
- 自动拼接
- 自动续上一条视频
- 一步到位的完整视频工作流

## 最短成功路径

### 1）准备 API Key
推荐环境变量：

```bash
export APIMART_API_KEY='sk-...'
export APIMART_BASE_URL='https://api.apimart.ai'
export APIMART_MODEL='doubao-seedance-2.0'
```

也支持放到：
- `runtime/.env.local`

### 2）检查连通性

```bash
python3 scripts/seedance2_video.py health
```

### 3）直接跑一条文生视频

```bash
python3 scripts/seedance2_video.py run \
  --prompt '雨夜街头，女生慢慢回头看镜头，电影感' \
  --duration 5 \
  --ratio 9:16
```

### 4）跑一条图生视频

```bash
python3 scripts/seedance2_video.py run \
  --prompt '让人物自然转头看向镜头，电影感' \
  --image /path/to/image.jpg \
  --duration 6 \
  --ratio 9:16
```

## CLI 命令

- `health`：检查 APIMart key / 余额 / base_url
- `plan`：只看意图理解与参数展开
- `create`：创建任务
- `status`：查询任务状态
- `wait`：轮询直到完成
- `download`：下载视频结果
- `upload-image`：上传本地图片到 APIMart
- `run`：一条龙执行 `create + wait + download`
- `current`：按 request_key / conversation_key 查当前任务或最近待回传任务
- `mark-sent`：标记该任务结果已成功回传到聊天

---

## 接入建议（很重要）

如果你是把它接进聊天系统或应用层，建议始终带上：

- `--request-key <message_id>`
- `--conversation-key <chat_id>`

这样能避免两类问题：
- 同一条消息反复下单
- 用户只是问进度，却被误判成重新生成

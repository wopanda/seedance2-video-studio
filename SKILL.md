# Seedance 2.0 Video Studio

> 状态：V1 可用，当前已接通 APIMart / doubao-seedance-2.0，可真实创建视频任务、轮询结果并下载视频。

这是一个给**自己用**的 Seedance 2.0 视频生成 skill。

它不是纯 prompt skill，也不是功能菜单。
它的目标是把现有能力做成一条更像产品的短路径：

1. 用户说创作意图
2. AI 判断这是“做一个 / 按参考做 / 接着改”中的哪一种
3. AI 识别素材角色、补默认参数、做三段式镜头规划
4. AI 调接口发起生成
5. AI 跟踪任务并返回结果，同时给出下一步可继续修改的入口

## 产品定位

一句话：
> 不是把它做成“更会写 prompt”，而是把它做成“更会替用户拍板并开工”的视频生成 skill。

核心原则：
- 用户只做高价值输入
- 复杂度渐进暴露
- 默认直接做，必要时才追问
- 保留创建任务 / 轮询 / 下载 / 返回结果的执行闭环

## 默认入口

### 入口 1：做一个
适合第一次生成。

用户可以直接说：
- 做一个 8 秒竖屏视频，雨夜街头，女生慢慢回头看镜头，电影感。
- 做一个电商广告，咖啡杯从静物变成热气腾腾的近景特写。

### 入口 2：按这个参考做
适合参考图 / 参考视频 / 参考音频驱动的生成或复刻。

用户可以直接说：
- 按这张图做一个人物转头视频。
- 按这个视频的镜头节奏，做一个类似的产品展示。
- 按这段音乐的感觉，做一个 8 秒 MV 感片段。

### 入口 3：接着上次改
适合延长、微调、重做、保一致性。

用户可以直接说：
- 接着上次那个视频，延长 4 秒。
- 人物不要变，镜头更稳一点。
- 保持这个风格，但改得更像短剧片头。

## V1 支持范围

- Text-only：纯文字生成
- First Frame：首帧图生
- First + Last Frame：首尾帧控制
- 轻量提示词扩展
- 任务创建 / 状态查询 / 结果下载
- APIMart 图片上传后的视频生成

暂不默认支持：
- 多段自动拼接
- 长视频自动拆段
- 自动生图
- 完整剧本与分镜流水线
- 稳定的飞书自动回传闭环

## 默认交互原则

### 1. 用户只说创作意图
用户默认不需要负责：
- 选 mode
- 写 asset mapping
- 写 negative constraints
- 写结构化 Seedance prompt
- 提交任务 / 查状态 / 下载结果

### 2. AI 负责内部规划与执行
AI 自动完成：
- 意图识别
- 素材角色识别
- 三段式镜头规划（开场 / 中段 / 落点）
- 一致性 / 节奏 / 风格 / 禁止项规划
- 默认参数补全
- 生成执行与结果回传

### 3. 只在必要时打断
只有以下情况才询问用户：
- 多张图主次不清
- 新生成 / 延长意图冲突
- 用户明确要求“先给我看方案 / prompt”
- 需求存在明显风格冲突
- 参考素材过多，系统无法判断优先级

## 执行规则

### A. 先判断用户意图，而不是先暴露 mode
优先识别为：
- 生成
- 复刻
- 延长
- 微调
- 补全
- 编辑

再在内部映射到具体生成路径。

### B. 再判断素材在本轮扮演什么角色
不要只看“有几张图”，而要看它们是不是：
- 主体锚点
- 首帧锚点
- 尾帧锚点
- 动作参考
- 风格参考
- 音频节奏参考
- 备用素材

### C. 做轻量提示词扩展
默认不是把句子写长，而是做结构化翻译：
- 补风格与镜头节奏
- 补时间结构（开场 / 中段 / 结尾）
- 补 negative constraints
- 补 generation settings

详细规则见：
- `references/interaction-spec-v1.md`
- `references/prompt-expansion-rules-v1.md`
- `references/result-card-spec-v1.md`

### D. 优先最短成功路径
默认目标是：
- 一次输入
- 一次规划判断
- 一次生成
- 一次返回结果

只有在关键歧义存在时，才进入确认流程。

## 输出格式

默认向用户返回：
1. 当前识别的意图 / 路径
2. 关键参数
3. task_id / 状态
4. 成功后的视频结果
5. 可继续操作的自然入口（如：再来一版 / 延长 / 更像广告 / 人物别变）
6. 若失败，给出简短可执行建议

## 设计原则

- 默认简单，内部专业
- 人定方向，AI定做法
- 不把专业术语压给用户
- 先保证单次生成闭环，再持续迭代到多段与拼接
- 先提升产品感，再扩能力面

## 当前迭代约束

V1 明确允许后续调整：
- 默认时长
- 默认画幅
- 提示词扩展力度
- 确认闸门阈值
- 结果卡返回样式
- 场景模板第一批（广告 / 人像 / 短剧 / 科普 / MV）

这不是一次写死的 skill，而是一个可持续进化的自用生成器。

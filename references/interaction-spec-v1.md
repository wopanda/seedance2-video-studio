# 交互规格 V1

## 目标
让用户尽可能用自然语言发起 Seedance 2.0 视频生成，而不是填写技术表单。

默认目标不是让用户先理解 mode，而是让用户直接表达创作意图。

## 默认入口

### 入口 1：做一个
适合首次生成。

示例：
> 做一个 8 秒竖屏视频，雨夜街头，女生慢慢回头看镜头，电影感。

### 入口 2：按这个参考做
适合按图片 / 视频 / 音频参考做。

示例：
> 按这张图做一个人物转头视频。
> 按这个视频的镜头节奏，做一个类似的产品展示。
> 按这段音乐的感觉，做一个 8 秒 MV 感片段。

### 入口 3：接着上次改
适合延长、微调、重做。

示例：
> 接着上次那个视频，延长 4 秒。
> 人物不要变，镜头更稳一点。
> 保持这个风格，但改得更像短剧片头。

## AI 默认行为
1. 先识别用户意图：生成 / 复刻 / 延长 / 微调 / 补全 / 编辑
2. 再识别素材角色：主体锚点 / 首帧锚点 / 尾帧锚点 / 动作参考 / 风格参考 / 音频节奏参考
3. 补默认参数
4. 做轻量提示词扩展
5. 执行生成
6. 返回结果卡

## 默认直接生成的条件
- 用户意图清楚
- 素材数量不多
- 主素材明确
- 没有明显模式冲突
- 不需要先让用户看 prompt / 方案

## 需要确认的条件
- 多张图主次不清
- 新生成 / 延长不明确
- 用户要求先看 prompt 或先看方案
- 风格和目标明显冲突
- 参考素材过多，系统无法判断优先级

## 内部规划规则

### 一、镜头规划
默认按三段式规划：
- 开场：建立场景 / 主体 / 起势
- 中段：动作 / 镜头推进 / 情绪变化
- 落点：构图停留 / 情绪收束 / 下一步衔接点

### 二、控制维度
默认考虑但不必全部展示给用户：
- 一致性：人物 / 产品 / 风格 / 首尾连续
- 节奏：镜头快慢 / 动作强度 / 卡点感
- 风格：广告 / 电影 / 短剧 / 仙侠 / 科普 / MV
- 禁止项：字幕 / watermark / logo / on-screen text

### 三、输出形态优先级
规划层只允许三种输出形态：
1. 直接生成
2. 先给简短方案卡再生成
3. 先给 prompt 预览（仅用户明确要求时）

默认优先级：
`直接生成 > 简短方案卡 > prompt 预览`

## 结果回传

### 创建任务时
- 当前识别的意图 / 路径
- 关键参数
- task_id
- 当前状态

### 成功后
- 视频结果
- 链接或文件
- 可继续操作的下一步建议，例如：
  - 再来一版
  - 延长 4 秒
  - 更像广告
  - 人物别变

### 失败后
- 失败原因
- 简短重试建议
- 如果是素材问题，要明确是“主素材不清”还是“参考不兼容”

返回格式细则见：`references/result-card-spec-v1.md`

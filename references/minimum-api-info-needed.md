# 接通 Seedance 2.0 Adapter 所需的最小接口信息

> 目标：不需要你一次给全量文档，只要给最小可接入信息，就能继续把 adapter 真接上。

## 最少需要这 5 类信息

### 1. 创建任务接口
至少给我其中一种：
- 官方 API 文档链接
- 接口截图
- 抓包截图
- curl 样例

我需要知道：
- 请求 URL
- 请求方法（POST/GET）
- 认证方式（Bearer / Cookie / Header）
- 请求体字段名

### 2. 查询任务状态接口
我需要知道：
- task_id 放哪里
- 查询 URL 长什么样
- 返回结果里状态字段叫什么
- 成功结果视频 URL 字段叫什么

### 3. 素材上传方式
如果 Seedance 2.0 不是直接吃本地文件，而要先上传素材，我需要知道：
- 图片 / 视频怎么上传
- 返回的素材字段是什么
- 首帧 / 尾帧 / 参考图如何在请求中引用

### 4. 模式字段怎么表达
我需要知道真实接口里：
- Text-only 是怎么传
- First Frame 是怎么传
- First + Last Frame 是怎么传
- Extend（后续）是怎么传

### 5. 一个真实成功样例
哪怕只给一个也行。

最有价值的是：
- 一次成功创建任务的请求样例
- 一次成功查询状态的返回样例

如果你已经在网页里生成过一次：
- 发我 network 截图

### 方案 C：给截图
即使没有完整文档，只要有：
- 创建任务页面参数截图
- 返回结果截图
- 状态页截图
也够我继续往下接。

# Seedance 2.0 自用生成 Skill 设计草案 V1

> 状态：迭代中（draft-v1）
> 目标：先把“自己真能用”的 Seedance 2.0 视频生成交互与提示词扩展规则定下来，后续可持续迭代到可执行 skill。

---

## 1. 产品目标

这是一个给**自己用**的 Seedance 2.0 视频生成 skill，不是商店展示版，也不是纯 prompt 工具。

它的目标是：

1. 用户用自然语言 + 可选素材发起视频需求
2. AI 自动判断最合适的 Seedance 2.0 模式
3. AI 自动做轻量结构化提示词扩展
4. AI 调用生成接口完成视频生成
5. AI 把结果回传到飞书

核心原则：
- **默认简单**：用户一句话也能开始
- **内部专业**：AI 内部按 Seedance 2.0 结构化逻辑处理
- **必要时确认**：只在关键歧义点打断用户
- **可持续迭代**：V1 先单条闭环，后续扩展多段/拼接/工作流

---

## 2. V1 范围

### V1 要做
- 文生视频（Text-only）
- 首帧图生视频（First Frame）
- 首尾帧视频（First + Last Frame）
- 生成前轻量提示词扩展
- 调用 Seedance 2.0 生成接口
- 轮询任务状态
- 下载 / 返回结果到飞书

### V1 不做
- 多段自动拼接
- 自动长视频拆段
- 自动生图
- 完整剧本/分镜工作台
- 重度导演型前置问答

这些能力后续进入 V2 / V3。

---

## 3. 交互原则（乔布斯式）

### 3.1 用户只做高价值输入
用户负责：
- 说清想做什么视频
- 提供必要素材（图/视频）
- 选择最终方向

用户不需要负责：
- 选 mode
- 写 asset mapping
- 写 negative constraints
- 写结构化 Seedance prompt
- 提交任务 / 查状态 / 下载结果

### 3.2 AI 负责结构翻译与执行
AI 自动完成：
- 模式判断
- 素材角色判断
- 轻量提示词扩展
- 参数补全
- 任务创建 / 轮询 / 返回

### 3.3 默认先给方案，不先逼问
除非存在关键歧义，否则：
- AI 直接判断模式
- AI 自动补默认参数
- AI 直接生成

### 3.4 复杂度渐进暴露
默认层：
- 一句话 + 素材 就能生成

高级层：
- 用户明确要看 prompt / 要专业模式 / 要更强控制时，再展开更多结构

---

## 4. 人机分工

### 人负责
- 创意目标
- 风格方向
- 素材选择（当素材冲突时）
- 最终拍板（继续 / 重做 / 延长）

### AI 负责
- 模式识别
- prompt 结构化扩展
- 基础镜头节奏补全
- negative constraints
- 生成参数默认值
- API 执行
- 结果回传

一句话：

> 人定方向，AI定做法。

## 5. 模式定义（V1）

### Mode A：Text-only
触发条件：
- 用户只给文字，没有图/视频素材

适用场景：
- 原创概念视频
- 氛围短片
- 简单广告 / 情绪镜头

### Mode B：First Frame
触发条件：
- 用户给 1 张图，且明显想以它为开场 / 主体锚点

适用场景：
- 角色一致性
- 产品首帧锚定
- 人像图生视频

### Mode C：First + Last Frame
触发条件：
- 用户给 2 张图，且意图明显是从 A 过渡到 B

适用场景：
- 状态转变
- 首尾构图明确
- 过渡型短视频

### 后续 V2 模式（先预留，不在 V1 实现）
- Extend
- All-Reference
- Multi-Segment

## 6. 默认交互流

### 6.1 文生
用户：
> 做一个 8 秒竖屏视频，雨夜街头，女生慢慢回头看镜头，电影感。

AI 内部：
- 判定 Text-only
- 自动补默认 ratio/duration/style
- 自动扩展为结构化 prompt
- 直接生成

### 6.2 首帧图生
用户：
> 用这张图做首帧，生成一个 6 秒视频，人物从静止到微笑转头。

AI 内部：
- 判定 First Frame
- `@image1` 作为首帧与主体锚点
- 自动补镜头节奏
- 直接生成

### 6.3 首尾帧
用户：
> 用图1做首帧，图2做尾帧，生成 6 秒视频，花朵从含苞到盛开。

AI 内部：
- 判定 First + Last Frame
- `@image1` = 起始锚点
- `@image2` = 结尾锚点
- 自动补过渡性 prompt
- 直接生成

## 7. 什么时候 AI 直接做，什么时候先问

### 7.1 直接做
满足以下任一组合时，默认不打断：
- 只有一句明确需求
- 只有一张图，且明显是首帧
- 两张图，且明显是首尾帧
- 风格 / 时长 /意图清楚

### 7.2 先问一句
出现以下情况时才确认：
- 多张图主次不清
- 用户意图不清：新生成 or 延长
- 风格冲突明显
- 用户说“先给我看 prompt”
- 用户说“按专业模式来”

确认问题应尽量短，只问关键决策，不做长问卷。

## 8. 自动提示词扩展设计逻辑

自动扩展不是“把句子写长”，而是做结构化翻译。

参考逻辑来源：
- Seedance 2.0 prompt-engineering 的结构化方法
- Shot Design 的镜头语言方法
- Seedance storyboard 工作流的时间轴/素材映射方法

### 扩展后的内部结构
1. Mode
2. Assets Mapping（如有素材）
3. Final Prompt
4. Negative Constraints
5. Generation Settings

---

## 9. 自动提示词扩展规则 V1

### 9.1 基础补全（默认自动做）
AI 自动补：
- mode
- duration 默认值
- ratio 默认值
- 基础风格词
- 基础镜头节奏
- 基础 negative constraints

### 9.2 结构化增强（默认轻量开启）
AI 自动把需求整理成：
- 开场（建立场景）
- 中段（动作/镜头推进）
- 结尾（情绪/构图落点）

### 9.3 高级拓展（仅显式触发）
只有用户明确说：
- 先给我看专业版提示词
- 帮我扩成分镜版
- 先不要生成，先出 prompt

才展开：
- 更完整时间码
- 更详细 assets mapping
- 更强 continuity / style control

---

## 10. 各模式的扩展模板

### 10.1 Text-only 模板
AI 内部扩展为：
- 风格/色调总纲
- 0-x 秒：建立场景
- x-y 秒：主体动作与镜头推进
- y-z 秒：收尾与落点
- Negative Constraints

### 10.2 First Frame 模板
AI 内部扩展为：
- `@image1` = first frame / identity anchor
- 开场保持与首帧一致
- 中段小动作 / 镜头运动
- 结尾给清晰落点
- Negative Constraints

### 10.3 First + Last Frame 模板
AI 内部扩展为：
- `@image1` = 起始锚点
- `@image2` = 结尾锚点
- 中段描述过渡方式
- 强调连续性与自然过渡
- Negative Constraints

---

## 11. 默认参数（可迭代）

V1 建议默认：
- duration：6s（可根据用户话术在 5/6/8s 间调整）
- ratio：9:16（若未说明，优先竖屏）
- style：电影感写实（若未说明）
- negative：no subtitles, no watermark, no logo, no on-screen text

这些默认值后续允许根据实际使用迭代。

---

## 12. 结果回传设计

### 12.1 飞书文字回执
返回：
- 使用模式
- 关键参数
- prompt 摘要
- task_id
- 当前状态

### 12.2 飞书媒体回执
成功后返回：
- 视频文件 / 视频链接
- 可选本地路径

失败后返回：
- 失败原因
- 下一步建议（缩短 prompt / 更简单素材 / 重试）

---

## 13. 可迭代点（必须保留）

这个 skill 明确支持持续迭代。

### 第一轮迭代关注
- 默认时长是否合适
- 默认 ratio 是否合适
- 自动扩展是否过度/不足
- 什么时候该确认、什么时候不该打断
- 返回飞书的信息是否过多或过少

### 后续迭代方向
- Extend
- 多段生成
- 自动拼接
- 更强多模态参考
- 草稿 / 正式模式分层
- 风格模板库
